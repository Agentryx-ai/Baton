# 이미지 9개 이상 선택 시 8개까지만 붙고 나머지가 조용히 누락됨

## 상태

- 상태: 미해결
- 발견일: 2026-07-22
- 우선순위: P2

## 증상

프로젝트리스 새 대화에서 서로 다른 지원 이미지 9개를 file chooser로 한 번에 선택하면
앞의 8개만 첨부되고 9번째 이미지는 아무 안내 없이 누락된다. 8개에 도달하면 첨부 버튼이
disabled 되지만 title과 accessible label은 여전히 단순히 `이미지 첨부`이며 최대 개수나
탈락한 파일을 설명하지 않는다.

재현 session:

```text
3b3ccf2a-677b-4eea-9b3d-8997ff00a4e0
```

지원하지 않는 MIME은 명확한 오류로 정상 차단됐고, 지원 이미지 2장의 동시 첨부·개별 제거·
전송·새로고침 보존도 정상이라 개수 초과 경로에 한정된 문제다.

## 원인

`addImageFiles()`는 남은 슬롯 수만큼 `Array.from(files).slice(0, available)`로 먼저 잘라낸다.
선택된 파일 수가 available보다 많은 경우를 검사하거나 사용자에게 알리는 경로가 없다.

## 영향

- 사용자는 선택한 모든 이미지가 전송된 것으로 오인할 수 있다.
- 비교·분석 요청에서 누락된 이미지 때문에 모델 답변이 불완전해진다.
- disabled 버튼만으로는 한도 도달 이유나 해결 방법을 알기 어렵다.

## 완료 조건

- 선택 파일이 남은 슬롯을 초과하면 최대 8개 제한과 누락 개수를 즉시 알린다.
- 첨부된 파일과 거부된 파일 이름을 구분해 확인할 수 있다.
- 8개 도달 시 버튼의 tooltip/accessible description에 한도 도달 이유를 표시한다.
- 0/7/8개 기존 첨부 상태에서 다중 선택하는 경계를 테스트한다.
