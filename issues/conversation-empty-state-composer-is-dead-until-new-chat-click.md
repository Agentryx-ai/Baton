# 대화 첫 진입의 빈 composer가 새 대화처럼 보이지만 전송할 수 없음

## 상태

- 상태: 미해결
- 발견일: 2026-07-22
- 우선순위: P2

## 증상

홈에서 `대화` 화면으로 들어왔고 선택된 session이 없을 때, 본문은 제목 `새 대화`와
composer를 표시한다. 사용자는 텍스트를 입력하고 이미지 첨부 미리보기·제거까지 할 수 있지만
상태는 `준비됨`이고 `메시지 보내기` 버튼은 계속 비활성이다.

좌측의 `새 대화` 버튼을 다시 클릭해야 상태가 `저장 전`으로 바뀌고 같은 텍스트·첨부를
정상 전송할 수 있다.

## 재현

1. Baton 홈을 연다.
2. 좌측 탐색에서 `대화`를 누른다.
3. session을 선택하거나 좌측 `새 대화`를 누르지 않은 채 빈 composer에 텍스트를 입력한다.
4. PNG를 첨부하고 preview와 제거 버튼이 나타나는지 확인한다.
5. `메시지 보내기` 버튼 상태를 확인한다.
6. 좌측 `새 대화`를 누르고 같은 입력을 다시 시도한다.

2026-07-22 복구 후 root와 독립 첨부 포맷 QA agent가 각각 재현했다. 첫 화면에서는 텍스트와
PNG preview가 있어도 전송할 수 없었고, 명시적으로 `새 대화`를 누르자 안내가
`일반 대화로 시작합니다. 파일 도구가 필요하면 폴더를 선택하세요.`로 바뀌며 즉시 전송
가능해졌다.

## 영향

- 첫 진입 화면이 실제 draft처럼 보여 사용자가 입력·첨부 작업을 시작하게 만든다.
- 전송할 수 없는 이유나 `새 대화`를 다시 눌러야 한다는 안내가 없다.
- 화면 전환 과정에서 입력 또는 첨부 선택을 반복하거나 잃을 수 있다.

## 완료 조건

- 선택 session이 없는 대화 첫 진입은 곧바로 유효한 `저장 전` draft를 만든다. 또는 composer를
  숨기고 하나의 명확한 `새 대화` CTA만 표시한다.
- 보이는 composer가 텍스트·첨부를 받는다면 유효한 model과 draft ID로 전송할 수 있다.
- 전송 불가 상태라면 입력과 첨부 상호작용도 비활성화하고 원인과 다음 동작을 설명한다.
- 홈→대화 첫 진입, reload, 모든 대화 삭제 후 진입을 포함하는 E2E가 통과한다.
